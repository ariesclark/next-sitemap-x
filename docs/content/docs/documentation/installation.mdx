### Step 1: Installing the package

Install next-sitemap-x via yarn with the following command

```bash
pnpm add next-sitemap-x
```

### Step 2: Creating `next-sitemap-x.config.js`

Create your `next-sitemap-x.config.js` file under your project root.

> âœ… `next-sitemap-x` will load environment variables from `.env` files by default.

```javascript
/** @type {import('next-sitemap-x').IConfig} */
module.exports = {
  siteUrl: process.env.SITE_URL || 'https://example.com',
  generateRobotsTxt: true, // (optional)
  // ...other options
}
```

### Step 3: Building sitemaps

Add next-sitemap-x as your postbuild script

```json
{
  "build": "next build",
  "postbuild": "next-sitemap-x"
}
```

### Custom config file

You can also use a custom config file instead of `next-sitemap-x.config.js`. Just pass `--config <your-config-file>.js` to build command (Example: [custom-config-file](https://github.com/ariesclark/next-sitemap-x/tree/master/examples/custom-config-file))

```json
{
  "build": "next build",
  "postbuild": "next-sitemap-x --config awesome.config.js"
}
```

### Building sitemaps with pnpm

When using pnpm you need to create a `.npmrc` file in the root of your project if you want to use a postbuild step:

```bash
//.npmrc
enable-pre-post-scripts=true
```
